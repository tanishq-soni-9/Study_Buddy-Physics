> **Capstone Project | Agentic AI Course 2026 | Dr. Kanthi Kiran Sirra**

## Problem Statement

**Domain:** Study Buddy for B.Tech Physics  
**User:** B.Tech students who need concept explanations, formula lookups, and numerical help at odd hours when no tutor is available.  
**Problem:** Students frequently struggle with physics concepts and formulas outside classroom hours. Existing resources require browsing multiple pages with no conversational interface. A 24/7 intelligent study assistant is needed that explains topics faithfully without hallucinating formulas or physical constants.  
**Success:** Agent answers 90%+ of curriculum questions faithfully (faithfulness ≥ 0.7), correctly refuses out-of-scope questions, uses a built-in physics calculator for numerical problems, and remembers the student's name across a multi-turn conversation.  
**Tool Used:** Physics formula calculator — the LLM extracts a mathematical expression from the question and evaluates it safely using Python's `math` module with no access to builtins.

---

## Architecture

```
User question
     ↓
[memory_node]    → append to history, sliding window (last 6), extract student name
     ↓
[router_node]    → LLM decides: retrieve / tool / memory_only
     ↓
[retrieval_node / tool_node / skip_node]
     ↓
[answer_node]    → system prompt + context + history → LLM response (grounded)
     ↓
[eval_node]      → faithfulness 0.0–1.0 → retry if < 0.7 (max 2 retries)
     ↓
[save_node]      → append answer to messages → END
```

**Key technologies:**
- **LangGraph** — StateGraph with 8 nodes, conditional edges, MemorySaver checkpointer
- **ChromaDB** — In-memory vector store with 12 B.Tech Physics documents
- **SentenceTransformer** (`all-MiniLM-L6-v2`) — query and document embeddings
- **Groq / LLaMA-3.3-70b** — LLM for routing, answering, and faithfulness evaluation
- **Streamlit** — Chat UI with session state, sidebar, and faithfulness display

## Setup & Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Set your Groq API key
```bash
export GROQ_API_KEY="your_groq_api_key_here"
```
Or create a `.env` file in the project root:
```
GROQ_API_KEY=your_groq_api_key_here
```

### 3. Run the notebook (exploration + RAGAS evaluation)
Open `day13_capstone.ipynb` in JupyterLab or VS Code and run all cells top-to-bottom:
- **Kernel → Restart & Run All**
- This generates `capstone_streamlit.py` and prints all test + RAGAS results

### 4. Launch Streamlit UI
```bash
streamlit run capstone_streamlit.py
```

### 5. Run pytest test suite
```bash
python -m pytest tests/test_agent.py -v
```

### 6. Run RAGAS evaluation standalone
```bash
python tests/test_agent.py
```
